管好房 App

大晟租賃 · 2024/12 — 2025/12

「管好房」是一款為不動產包租代管租賃公司客戶設計的 App，服務對象為該客戶內部的業務、管理者、房東、房客。主要想解決的問題是減少業務、房東、房客三方間的溝通成本、合約管理與租金繳納等痛點。我負責接手前任開發者的架構，進行全面重構——從功能模組、使用者操作流程（User Flow）到 UI 介面，都重新設計與實作，最終完成雙平台上架。

核心挑戰與解決方式
01

遺留架構重構

接手的程式碼架構混亂，邏輯耦合度高，難以維護與擴充新功能。
重新梳理程式碼邏輯，根據新需求重新規劃資料結構與模組化架構，提升可維護性與執行效率。
02

跨團隊溝通磨合

初期對「需求到交付」的流程不熟悉，與後端的對接效率偏低。
主動建立溝通機制，和後端工程師確立 API 規格與交付標準，逐步磨合出高效的協作流程。
03

面對大量重構工作，需要在有限時間內確保品質與效率兼顧。
導入 Cursor 與 Claude 輔助編碼、需求分析與除錯，顯著縮短開發週期並提升代碼品質。

TTE

TimeTree Exporter

潛水業者內部工具

為一間潛水業者打造的內部工具。該業者使用 TimeTree 進行內部排班管理，但 TimeTree 無法直接匯出排班資料。此工具的目的是將 TimeTree 的排班資料同步並匯出為 CSV 檔，方便業者計算薪資。

核心挑戰與解決方式
01

資料取得限制

TimeTree 無法直接匯出排班資料，且前端受 CORS 限制無法直接呼叫 API。
透過 Node.js 後端作為代理，呼叫 TimeTree Web API 取得排班資料並同步至資料庫。
02

效能優化

初版使用 Puppeteer 爬蟲，3000 筆事件需耗時 2-3 小時。
重構為 API 批次取得（每次 300 筆），同樣資料量僅需 30 秒，效能提升超過 200 倍。
03

全端獨立開發

從需求分析到部署上線皆獨立完成，需掌握前後端與雲端部署。
前端 Flutter Web、後端 Node.js、資料庫 Supabase、部署 GCP Cloud Run + Firebase Hosting，獨立完成全流程。
技術使用

Flutter Web

前端介面開發，提供同步觸發、時間範圍篩選、CSV 匯出下載與匯出歷史查詢功能。

FlutterDart

GCP + Node.js

Node.js 後端部署於 GCP Cloud Run，負責代理呼叫 TimeTree 內部 API 並將資料同步至資料庫。前端靜態檔案託管於 Firebase Hosting。

GCP Cloud RunFirebase HostingNode.js

Supabase

使用者認證（Supabase Auth）、事件資料儲存（PostgreSQL）、Row Level Security 確保資料隔離。
